# Memory Bank

The Memory Bank is the current operating memory for this project. It answers: **Where are we right now?**

## Files

| File | Purpose |
|------|---------|
| `projectbrief.md` | What this project is, its purpose, and success criteria |
| `productContext.md` | Users, workflows, goals, UX expectations |
| `activeContext.md` | Current focus, project state, blockers, next steps |
| `systemPatterns.md` | Architecture patterns, module boundaries, data flow |
| `techContext.md` | Languages, frameworks, tools, commands, deployment |
| `progress.md` | Completed work, remaining work, known issues |
| `decisions.md` | Dated decision records with rationale |
| `handoff.md` | Read order, safety posture, immediate next action |

## Usage

Before starting non-trivial work, read:
1. `projectbrief.md`
2. `activeContext.md`
3. `progress.md`

After significant work, update:
1. `activeContext.md` — current focus, recent changes, blockers
2. `progress.md` — completed work, remaining work
3. `decisions.md` — major technical or product decisions

## Rules

- Keep files short, current, and actionable
- Do not duplicate Graphify output here — link to it instead
- Do not use as transcript dump
- Do not store secrets, credentials, or raw private data
